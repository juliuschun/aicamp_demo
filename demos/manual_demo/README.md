# 실습: 회의록 → 문서 자동 생성

`raw_data/` 안에 가상 회사 자료가 있습니다 (회의록 10개 + 슬랙 로그 3개).

이걸 Claude Code에게 분석시켜 문서를 만들어봅니다.

---

## 실습 1: 온보딩 가이드

```
raw_data 폴더의 회의록과 슬랙 로그를 분석해서
신입사원용 온보딩 가이드를 만들어줘.

포함: 회사 소개, 주요 용어, 팀별 역할, 커뮤니케이션 채널, 회사 문화
```

## 실습 2: FAQ

```
raw_data의 모든 파일을 분석해서 FAQ를 만들어줘.
슬랙에서 자주 나온 질문, 회의에서 반복 논의된 주제 기반으로.
```

## 실습 3: 업무 프로세스 (SOP)

```
raw_data를 분석해서 업무 프로세스 매뉴얼을 만들어줘.
채용, 제품 기획, 고객 대응, 의사결정 프로세스를 단계별로 정리.
```

---

## 도전 과제

- 개발팀 전용 가이드 (dev-issues 채널 중심)
- 경영진 보고서 (주요 의사결정 요약)
- 제품 히스토리 (TaskFlow 개발 타임라인)

---

**팁**: 프롬프트가 구체적일수록 결과물이 좋아집니다.
