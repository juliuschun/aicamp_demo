# Demo 1: Raw 정보에서 매뉴얼 자동 생성

> "산재된 정보를 체계적인 문서로 변환하는 AI의 힘을 경험하세요"

## 데모 개요

### 목표
- 회의록, 슬랙 로그 등 비정형 데이터를 분석하여 체계적인 문서로 변환
- Claude Code의 파일 분석 및 문서 생성 능력 시연
- 비개발자도 쉽게 따라할 수 있는 AI 활용법 학습

### 핵심 메시지
기업에는 수많은 정보가 여기저기 흩어져 있습니다. 회의록, 슬랙 대화, 이메일...
이런 **산재된 정보**를 일일이 정리하는 것은 엄청난 시간이 소요됩니다.

Claude Code를 사용하면:
- 10개의 회의록 + 3개의 슬랙 채널 = **수백 페이지의 정보**
- 이것을 단 **몇 분 만에** 체계적인 문서로 변환할 수 있습니다

## 라이브 데모 스크립트

### 사전 준비 (데모 전)
1. 터미널을 열고 `manual_demo` 폴더로 이동
2. Claude Code 실행 확인: `claude` 명령어 입력

---

### Step 1: 문제 제시 (2분)

#### 진행자 멘트
> "여러분, 새로운 직원이 입사했다고 가정해볼게요.
> 이 직원에게 우리 회사에 대해 알려주려면 어떻게 해야 할까요?
>
> 보통은 이렇게 하죠:
> - 회의록 찾아보기
> - 슬랙 히스토리 뒤지기
> - 선배한테 물어보기
>
> 시간도 오래 걸리고, 정보도 불완전하죠.
>
> 지금부터 이 문제를 Claude Code로 해결해보겠습니다."

#### 데모 액션
터미널에서 raw 데이터를 보여줍니다:

```bash
# 회의록 목록 확인
ls raw_data/회의록/

# 슬랙 로그 목록 확인
ls raw_data/슬랙로그/

# 회의록 하나 열어보기 (내용이 많음을 보여줌)
head -50 raw_data/회의록/2024-01-신제품기획.md
```

#### 진행자 멘트
> "보시는 것처럼 회의록 10개, 슬랙 채널 3개...
> 이걸 다 읽고 정리하려면 최소 반나절은 걸리겠죠?
>
> Claude Code에게 맡겨봅시다."

---

### Step 2: 온보딩 가이드 생성 (5분)

#### 진행자 멘트
> "먼저 신입사원을 위한 온보딩 가이드를 만들어보겠습니다.
> Claude Code를 실행하고, 프롬프트를 입력합니다."

#### 데모 액션
```bash
# Claude Code 실행
claude
```

Claude Code에 다음 프롬프트 입력:

```
raw_data 폴더에 있는 회의록과 슬랙 로그를 전부 분석해서
신입사원을 위한 온보딩 가이드를 만들어줘.

다음 내용을 포함해줘:
1. 회사 소개 (제품, 비전, 팀 구성)
2. 주요 용어 및 약어 정리
3. 팀별 역할과 담당자
4. 커뮤니케이션 채널 가이드
5. 알아두면 좋은 회사 문화

output/onboarding_guide.md로 저장해줘.
```

#### 예상 소요 시간
- 파일 분석: 30초 ~ 1분
- 문서 생성: 1분 ~ 2분

#### 진행자 멘트 (생성 중)
> "Claude Code가 지금 13개 파일을 분석하고 있습니다.
> 각 회의록에서 언급된 팀, 프로젝트, 의사결정 사항들을 추출하고,
> 슬랙 로그에서 회사 문화와 커뮤니케이션 패턴을 파악하고 있어요."

#### 결과 확인
```bash
# 생성된 파일 확인
cat output/onboarding_guide.md
```

---

### Step 3: FAQ 생성 (3분)

#### 진행자 멘트
> "온보딩 가이드가 완성됐네요!
> 이번엔 자주 묻는 질문(FAQ)을 만들어볼까요?
> 슬랙에서 반복적으로 나온 질문들을 추출해봅시다."

#### 데모 액션
Claude Code에 프롬프트 입력:

```
raw_data 폴더의 모든 파일을 분석해서 FAQ 문서를 만들어줘.

슬랙 로그에서 자주 나온 질문, 회의록에서 반복 논의된 주제를 기반으로
실제 구성원들이 궁금해할 만한 질문과 답변을 정리해줘.

카테고리별로 구분하고, 각 답변에는 관련 맥락도 포함해줘.

output/faq.md로 저장해줘.
```

#### 진행자 멘트 (생성 중)
> "FAQ는 단순히 질문-답변이 아닙니다.
> Claude는 문맥을 이해해서, 왜 그런 질문이 나왔는지,
> 어떤 배경이 있는지까지 파악해서 답변을 구성합니다."

---

### Step 4: SOP(업무 프로세스) 생성 (3분)

#### 진행자 멘트
> "마지막으로 업무 프로세스 매뉴얼을 만들어보겠습니다.
> 회의록에 나온 의사결정 과정, 승인 절차 등을 추출해서
> 표준 업무 프로세스(SOP)로 정리합니다."

#### 데모 액션
Claude Code에 프롬프트 입력:

```
raw_data 폴더의 회의록과 슬랙 로그를 분석해서
업무 프로세스 매뉴얼(SOP)을 만들어줘.

다음 업무별로 프로세스를 정리해줘:
1. 채용 프로세스 (면접, 평가, 온보딩)
2. 제품 기획 프로세스 (아이디어 → 개발 → 출시)
3. 고객 대응 프로세스 (문의 접수 → 해결 → 피드백)
4. 의사결정 프로세스 (제안 → 검토 → 승인)

각 프로세스는 단계별로, 담당자와 함께 작성해줘.

output/sop.md로 저장해줘.
```

## FAQ

### Q: Claude Code가 설치되지 않아요
**A:** Node.js가 설치되어 있는지 확인하세요.
```bash
node --version  # v18 이상 권장
npm --version
```
Node.js가 없다면 https://nodejs.org/ 에서 설치하세요.

### Q: API 키 에러가 발생해요
**A:** 환경변수가 제대로 설정되었는지 확인하세요.
```bash
echo $ANTHROPIC_API_KEY
```
값이 출력되지 않으면 환경변수 설정이 필요합니다.

### Q: 생성된 문서가 영어로 나와요
**A:** 프롬프트에 "한국어로 작성해줘"를 추가하세요.
```
... 온보딩 가이드를 한국어로 만들어줘.
```

### Q: 분석에 시간이 너무 오래 걸려요
**A:** 파일 수가 많으면 시간이 더 걸릴 수 있습니다.
특정 폴더만 분석하도록 범위를 좁혀보세요.
```
raw_data/회의록 폴더만 분석해서...
```

### Q: 결과물의 품질을 높이려면?
**A:** 프롬프트를 더 구체적으로 작성하세요.
- 원하는 섹션 명시
- 포함할 내용과 제외할 내용 명시
- 결과물의 형식(표, 목록 등) 지정
- 대상 독자 명시 (신입사원, 경영진 등)

## 트러블슈팅

### "Permission denied" 에러
```bash
# output 폴더에 쓰기 권한 확인
ls -la output/

# 권한 부여 (필요시)
chmod 755 output/
```

### "File not found" 에러
```bash
# 현재 위치 확인
pwd

# manual_demo 폴더에 있는지 확인
ls raw_data/
```

### Claude Code가 멈춘 것 같을 때
- Ctrl+C로 중단 후 다시 시작
- 프롬프트를 더 간단하게 수정하여 재시도

### 결과물이 잘리는 경우
긴 문서는 여러 번에 나눠서 생성:
```
먼저 1~3번 섹션만 작성해줘.
```
그 다음:
```
이어서 4~6번 섹션을 작성해줘.
```
