## 안건 1: TaskFlow 2.0 핵심 기능 방향성

**김민수**: 작년 말 고객 인터뷰 결과를 보면 실시간 협업 기능에 대한 요청이 가장 많았어요. 2.0에서는 이 부분을 중점적으로 개선하는 게 어떨까요?

**박지현**: 동의합니다. 현재 아키텍처로는 실시간 동기화가 어려워서 WebSocket 기반으로 재설계가 필요해요. 개발 기간이 꽤 걸릴 것 같은데, 최소 3개월은 잡아야 할 것 같습니다.

**이서연**: 경쟁사 분석 결과도 공유드리면, Asana와 Monday가 이미 실시간 협업을 제공하고 있고, 우리 고객 중 이탈한 케이스의 40%가 이 기능 부재를 언급했어요.

**정우성**: 기술적으로 보면, 먼저 코멘트 기능부터 실시간으로 전환하고, 그 다음 태스크 상태 변경, 마지막으로 동시 편집 순서로 가는 게 리스크 관리 측면에서 좋을 것 같습니다.

**한소희**: UX 관점에서 실시간 기능이 추가되면 사용자에게 누가 어디를 보고 있는지 표시하는 Presence 기능도 필요해요. 디자인 리서치 결과 이게 협업감을 크게 높여준다고 합니다.

### 결정 사항
- TaskFlow 2.0의 핵심 테마: "실시간 협업 플랫폼"으로 확정
- 개발 우선순위: 실시간 코멘트 → 태스크 동기화 → Presence 기능 → 동시 편집
- 출시 목표: 2024년 7월 (6개월 개발 기간)

### 액션 아이템
- [ ] 박지현: WebSocket 아키텍처 설계안 작성 (1/22까지)
- [ ] 이서연: 상세 기능 명세서 작성 (1/25까지)
- [ ] 한소희: Presence 기능 UI/UX 프로토타입 (1/29까지)
- [ ] 정우성: 개발팀 리소스 플래닝 (1/22까지)

## 안건 2: 기술 스택 검토

**박지현**: 실시간 기능을 위해 Socket.io를 도입하려고 합니다. 서버 사이드는 현재 Node.js를 쓰고 있어서 호환성은 문제없어요.

**정우성**: Redis Pub/Sub도 함께 도입해야 할 것 같은데, 인프라 비용이 월 $500 정도 추가될 것으로 예상됩니다.

**김민수**: 비용은 감당 가능합니다. 다만 보안 측면에서 문제는 없는지 확인이 필요해요.

**박지현**: 네, 보안팀과 별도로 검토 예정입니다. 특히 엔터프라이즈 고객들의 요구사항을 고려해서 E2E 암호화도 검토하고 있어요.

### 결정 사항
- Socket.io + Redis Pub/Sub 스택 채택
- 보안 검토 후 최종 확정 (2월 중)

### 액션 아이템
- [ ] 박지현: 보안 검토 일정 조율 (1/19까지)
- [ ] 정우성: Redis 인프라 구축 PoC (2/5까지)
