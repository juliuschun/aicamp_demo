## 안건 1: 1월 고객 피드백 종합 분석

**최영호**: 1월 한 달간 접수된 피드백을 정리해왔습니다. 총 156건 중 기능 요청이 67건, 버그 리포트가 42건, 불만 사항이 47건이었어요.

**강다희**: 불만 사항 중에서 가장 많았던 건 "로딩 속도가 느리다"는 의견이에요. 특히 대시보드 페이지에서 태스크가 100개 이상인 프로젝트에서 체감이 심하다고 합니다.

**정우성**: 아, 그 이슈 저희도 인지하고 있어요. 페이지네이션을 적용하면 해결되는데, 2.0 개발하면서 같이 처리하려고 했거든요. 근데 불만이 이렇게 많으면 우선 처리가 필요할 것 같네요.

**이서연**: 2.0 로드맵에 영향 없이 처리 가능할까요?

**정우성**: 2주 정도면 될 것 같은데, 스프린트 중간에 끼워넣으면 가능합니다.

**윤재석**: 영업팀 입장에서도 빨리 해결되면 좋겠어요. A사가 이 문제로 계약 갱신을 미루고 있거든요. 작년 ARR의 8% 정도 되는 큰 고객이에요.

### 결정 사항
- 대시보드 로딩 속도 개선을 2월 스프린트에 긴급 배치
- 해결 후 A사에 별도 안내 및 계약 갱신 재논의

### 액션 아이템
- [ ] 정우성: 페이지네이션 적용 개발 (2/22까지)
- [ ] 윤재석: A사 담당자에게 개선 일정 공유 (2/9까지)
- [ ] 최영호: 속도 이슈 관련 문의 고객 리스트 정리 (2/12까지)

## 안건 2: 기능 요청 우선순위 검토

**이서연**: 기능 요청 67건을 카테고리별로 분류했어요. 1위가 "Slack 연동 강화" 23건, 2위가 "간트 차트" 18건, 3위가 "타임 트래킹" 12건입니다.

**최영호**: Slack 연동은 현재도 있긴 한데, 알림만 가능하잖아요. 고객들이 원하는 건 Slack에서 직접 태스크를 생성하거나 상태를 변경하는 거예요.

**정우성**: Slack API 활용하면 충분히 가능한데, 2.0 개발 후에 진행하는 게 좋을 것 같아요. 지금 리소스가 빠듯해서요.

**이서연**: 그럼 Q3에 Slack 연동 강화를 배치하고, 간트 차트는 2.0에 포함시키는 걸로 할까요? 간트 차트가 엔터프라이즈 고객들한테 특히 중요하다고 들었어요.

**윤재석**: 맞아요. 엔터프라이즈 영업할 때 간트 차트 없냐고 많이 물어봅니다. 경쟁 PT할 때 불리하게 작용하고 있어요.

### 결정 사항
- 간트 차트: TaskFlow 2.0에 포함 (7월 출시)
- Slack 연동 강화: Q3 (9월 목표)
- 타임 트래킹: Q4 검토

### 액션 아이템
- [ ] 이서연: 간트 차트 기능 명세 추가 작성 (2/15까지)
- [ ] 정우성: 간트 차트 개발 공수 산정 (2/16까지)

## 안건 3: NPS 점수 분석

**강다희**: 1월 NPS 조사 결과가 나왔는데, 점수가 32점으로 전월 대비 5점 하락했어요.

**이서연**: 어떤 부분에서 하락이 컸나요?

**강다희**: "다른 사람에게 추천하겠다"는 항목에서 중립 응답이 늘었어요. 구체적 이유를 물었을 때 "기능은 좋은데 최근 업데이트가 없다"는 의견이 많았습니다.

**최영호**: 2.0 개발에 집중하느라 소소한 업데이트가 없었던 게 영향을 준 것 같아요. 작은 기능이라도 정기적으로 릴리즈하는 게 필요할 것 같습니다.

**이서연**: 좋은 지적이에요. 월간 소규모 업데이트 계획을 세워보겠습니다.

### 결정 사항
- 월 1회 소규모 기능 업데이트 릴리즈 시작 (3월부터)
- NPS 개선 목표: Q2까지 40점 회복
