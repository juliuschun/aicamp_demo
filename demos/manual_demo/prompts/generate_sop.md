# SOP(업무 프로세스 매뉴얼) 생성 프롬프트

표준 업무 프로세스(Standard Operating Procedure) 문서를 생성하는 프롬프트 템플릿입니다.

---

## 기본 프롬프트

가장 간단한 형태의 프롬프트입니다.

```
raw_data 폴더에 있는 회의록과 슬랙 로그를 분석해서
업무 프로세스 매뉴얼(SOP)을 만들어줘.

회의록에서 언급된 프로세스, 절차, 승인 체계를 찾아서 정리해줘.

output/sop.md로 저장해줘.
```

---

## 고급 프롬프트 (더 상세한 결과)

체계적이고 실행 가능한 SOP를 원할 때 사용하세요.

```
raw_data 폴더의 회의록 10개와 슬랙 로그 3개를 분석해서
종합 업무 프로세스 매뉴얼(SOP)을 만들어줘.

## 분석 대상 프로세스

### 1. 채용 프로세스
- 채용 공고 → 서류 심사 → 면접 → 최종 결정 → 온보딩
- 각 단계별 담당자, 소요 시간, 체크리스트

### 2. 제품 기획 및 개발 프로세스
- 아이디어 제안 → 기획 검토 → 개발 → 베타 테스트 → 출시
- 의사결정 포인트, 승인권자

### 3. 고객 대응 프로세스
- 문의 접수 → 분류 → 처리 → 피드백 → 종료
- 에스컬레이션 기준, SLA

### 4. 의사결정 프로세스
- 제안 → 검토 → 논의 → 결정 → 실행
- 결정 권한 매트릭스

## SOP 형식 요구사항

각 프로세스마다:
1. **목적**: 이 프로세스가 왜 필요한지
2. **범위**: 언제, 누구에게 적용되는지
3. **단계별 절차**: 순서도 형태로
4. **담당자**: 각 단계의 책임자
5. **산출물**: 각 단계에서 만들어지는 결과물
6. **예외 상황**: 일반적이지 않은 케이스 처리법
7. **관련 도구/채널**: 사용하는 시스템이나 커뮤니케이션 채널

output/sop.md로 저장해줘.
```

## 예상 결과물 형식

Claude Code가 생성하는 SOP 문서의 예상 형식입니다.

```markdown
# TechStartup Inc. 업무 프로세스 매뉴얼 (SOP)

> 본 문서는 주요 업무 프로세스의 표준 절차를 정의합니다.
> 최종 수정일: 2024-10-01

---

## 1. 채용 프로세스

### 1.1 목적
우수 인재를 신속하고 효율적으로 채용하기 위한 표준 절차

### 1.2 범위
모든 정규직 채용에 적용 (인턴, 계약직은 별도 프로세스)

### 1.3 프로세스 개요

```
채용 요청 → 공고 게시 → 서류 심사 → 원데이 면접 → 최종 결정 → 오퍼 → 온보딩
   (1일)      (상시)      (3일)       (1일)        (2일)      (3일)    (D+1)
```

### 1.4 단계별 상세 절차

#### Step 1: 채용 요청
| 항목 | 내용 |
|------|------|
| 담당자 | 채용 요청 팀장 |
| 승인권자 | CEO |
| 산출물 | 채용 요청서 (JD 초안 포함) |
| 소요시간 | 1일 |

**절차:**
1. 팀장이 채용 필요성 및 JD 초안 작성
2. HR팀에 채용 요청서 제출
3. CEO 승인 후 공고 게시 진행

#### Step 2: 공고 게시
| 항목 | 내용 |
|------|------|
| 담당자 | HR팀 |
| 채널 | 원티드, 로켓펀치, 링크드인 |
| 산출물 | 채용 공고 URL |

#### Step 3: 서류 심사
| 항목 | 내용 |
|------|------|
| 담당자 | HR팀 + 해당 팀장 |
| 기준 | 경력, 기술 스택, 포트폴리오 |
| 소요시간 | 지원 후 3일 이내 결과 통보 |

#### Step 4: 원데이 면접
| 항목 | 내용 |
|------|------|
| 담당자 | 해당 팀장, CTO, CEO |
| 구성 | 기술면접(1시간) + 라이브코딩(2시간) + 컬처핏(30분) |
| 장소 | 회의실 A 또는 화상 |

**면접 가이드:**
- 기술면접: 팀장/CTO가 기술 역량 검증
- 라이브코딩: 과제 전형 대체, 실시간 문제 해결
- 컬처핏: CEO 참석, 가치관 및 문화 적합성 확인

#### Step 5: 최종 결정
| 항목 | 내용 |
|------|------|
| 참석자 | 면접관 전원 |
| 방식 | 면접 직후 또는 익일 회의 |
| 결정사항 | 합격/불합격, 연봉 범위 |

#### Step 6: 오퍼
| 항목 | 내용 |
|------|------|
| 담당자 | HR팀 |
| 포함내용 | 연봉, 스톡옵션, 입사일, 복리후생 |
| 협상범위 | 연봉 최대 9천만원, 스톡옵션 0.3% |

### 1.5 예외 상황

**시니어급 급채용 시:**
- 레퍼럴 보너스 200만원 (일반 100만원)
- 면접 일정 우선 조율
- 연봉 협상 유연하게 대응

### 1.6 관련 채널
- 채용 논의: HR팀 DM
- 면접 일정: Google Calendar
- 지원자 관리: 채용 관리 시스템

## 커스터마이징 팁

### 1. 상세 수준 조절
```
- "핵심 단계만 간략하게"
- "각 단계마다 구체적인 체크리스트 포함"
- "스크린샷/예시 화면 설명 추가"
```

### 2. 형식 지정
```
- "순서도(플로우차트) 형태로"
- "표 중심으로 정리"
- "체크리스트 형태로"
```

### 3. 특정 역할 관점
```
- "신입사원이 따라할 수 있도록"
- "관리자 승인 중심으로"
- "외부 감사 대응용으로"
```

### 4. 보완 정보 추가
```
- "예외 상황 처리 방법 상세히"
- "자주 하는 실수와 방지법"
- "개선이 필요한 부분 제안"
```

### 5. 시각적 요소
```
- "각 단계를 아이콘으로 구분"
- "담당자별 색상 구분"
- "타임라인 형태로"
```

- 프로세스명
- 한 줄 설명
- 5단계 이내 핵심 절차
- 담당자
- 주의사항 1개

벽에 붙여놓고 볼 수 있게 간결하게.

output/sop_onepage.md로 저장해줘.
```

### 예시 2: 역할별 가이드
```
raw_data를 분석해서 역할별 SOP 가이드를 만들어줘.

각 역할이 "언제, 무엇을, 어떻게" 해야 하는지 정리.

output/sop_by_role.md로 저장해줘.
```

### 예시 3: 문제 상황 대응 SOP

output/sop_with_improvement.md로 저장해줘.
