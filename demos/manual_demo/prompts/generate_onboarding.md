# 온보딩 가이드 생성 프롬프트

신입사원을 위한 온보딩 가이드를 생성하는 프롬프트 템플릿입니다.

---

## 기본 프롬프트

가장 간단한 형태의 프롬프트입니다. 빠르게 결과를 얻고 싶을 때 사용하세요.

```
raw_data 폴더에 있는 회의록과 슬랙 로그를 분석해서
신입사원을 위한 온보딩 가이드를 만들어줘.

output/onboarding_guide.md로 저장해줘.
```

---

## 고급 프롬프트 (더 상세한 결과)

더 체계적이고 상세한 온보딩 가이드를 원할 때 사용하세요.

```
raw_data 폴더에 있는 회의록 10개와 슬랙 로그 3개를 전부 분석해서
신입사원을 위한 종합 온보딩 가이드를 만들어줘.

## 포함할 내용

### 1. 회사 개요
- 우리가 만드는 제품/서비스 소개
- 회사의 미션과 비전
- 최근 주요 성과 및 마일스톤

### 2. 조직 구조
- 팀별 구성과 역할
- 주요 담당자 및 연락처
- 보고 체계

### 3. 용어 및 약어 사전
- 회사에서 자주 사용하는 용어
- 프로젝트명, 제품명 약어
- 기술 용어 설명

### 4. 커뮤니케이션 가이드
- 슬랙 채널별 용도
- 회의 문화 및 규칙
- 의사결정 프로세스

### 5. 회사 문화
- 업무 스타일
- 복리후생 및 사내 이벤트
- 알아두면 좋은 팁

### 6. 첫 주 체크리스트
- 입사 첫날 해야 할 일
- 첫 주 동안 완료할 사항
- 30일 내 목표

## 작성 스타일
- 친근하고 읽기 쉬운 톤
- 구체적인 예시 포함
- 중요한 정보는 강조 표시

output/onboarding_guide.md로 저장해줘.
```

---

## 팀별 맞춤 프롬프트

특정 팀 신입사원을 위한 맞춤 가이드가 필요할 때 사용하세요.

### 개발팀용
```
raw_data 폴더를 분석해서 개발팀 신입사원 전용 온보딩 가이드를 만들어줘.

특히 다음 내용에 집중해줘:
- 기술 스택과 아키텍처
- 개발 프로세스 (스프린트, 코드 리뷰 등)
- 주요 프로젝트 히스토리
- 개발팀 관련 슬랙 채널 사용법
- 기술 의사결정 참고 자료

output/dev_onboarding.md로 저장해줘.
```

### 영업/CS팀용
```
raw_data 폴더를 분석해서 영업/CS팀 신입사원 전용 온보딩 가이드를 만들어줘.

특히 다음 내용에 집중해줘:
- 제품 기능과 강점
- 고객 유형과 주요 고객사
- 자주 묻는 질문과 답변
- 고객 대응 프로세스
- 경쟁사 대비 차별점

output/sales_cs_onboarding.md로 저장해줘.
```

## 예상 결과물 형식

Claude Code가 생성하는 온보딩 가이드의 예상 형식입니다.

```markdown
# TechStartup Inc. 신입사원 온보딩 가이드

> 환영합니다! 이 가이드는 여러분이 회사에 빠르게 적응할 수 있도록
> 핵심 정보를 정리한 문서입니다.

## 1. 회사 소개

### 우리가 만드는 것
TaskFlow는 팀 협업과 프로젝트 관리를 위한 SaaS 플랫폼입니다...

### 최근 성과
- 2024년 3월: TaskFlow 2.0 런칭 (실시간 협업 기능)
- 2024년 9월: MAU 10,000 돌파
- ...

## 2. 팀 소개

| 팀 | 역할 | 주요 담당자 |
|---|---|---|
| 개발팀 | 제품 개발 및 유지보수 | 박지현(CTO), 정우성(팀장) |
| 영업팀 | 고객 확보 및 관리 | 이수진(팀장) |
| ...

## 3. 용어 사전

- **TaskFlow**: 우리 회사의 핵심 제품명
- **스프린트**: 2주 단위 개발 주기
- ...

## 4. 커뮤니케이션

### 슬랙 채널
- `#general`: 전사 공지 및 일상 대화
- `#dev-issues`: 개발 관련 이슈 논의
- ...

## 5. 첫 주 체크리스트

- [ ] 슬랙 채널 가입하기
- [ ] 팀원들과 인사하기
- [ ] 제품 데모 받기
- ...
```

## 커스터마이징 팁

### 1. 대상 독자 명시
프롬프트에 대상을 명확히 하면 더 적절한 결과를 얻을 수 있습니다.
```
"개발 경험이 없는 비개발 직군 신입사원"을 위한...
"5년차 이상 시니어 개발자"를 위한...
```

### 2. 톤앤매너 지정
```
- "친근하고 캐주얼한 톤으로"
- "공식적이고 격식있는 문체로"
- "간결하고 핵심만"
```

### 3. 분량 조절
```
- "A4 2페이지 분량으로"
- "각 섹션 200자 이내로"
- "핵심 정보만 간략하게"
```

### 4. 특정 정보 강조
```
- "특히 보안 관련 내용을 상세히"
- "원격 근무 가이드를 포함해서"
- "회의 문화를 중점적으로"
```

### 5. 형식 지정
```
- "표 형식으로 정리해줘"
- "체크리스트 형태로"
- "Q&A 형식으로"
```

## 실전 예시

### 예시 1: 스타트업 분위기 강조
```
raw_data를 분석해서 온보딩 가이드를 만들어줘.
우리 회사는 스타트업이라 자유롭고 수평적인 문화야.
그 분위기가 느껴지도록 친근한 톤으로 작성해줘.
이모지는 사용하지 말고, 핵심 정보 위주로 정리해줘.
```

### 예시 2: 빠른 온보딩 강조
```
raw_data를 분석해서 "30분 만에 읽는 온보딩 가이드"를 만들어줘.
가장 중요한 정보만 압축해서 정리하고,
나머지는 "더 알아보기" 섹션에 참고 링크로 정리해줘.
```

### 예시 3: 체크리스트 중심
```
raw_data를 분석해서 온보딩 체크리스트를 만들어줘.
첫째 날, 첫 주, 첫 달로 구분하고,
각 단계에서 완료해야 할 태스크를 체크리스트로 정리해줘.
```
