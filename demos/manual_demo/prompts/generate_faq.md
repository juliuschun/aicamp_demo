# FAQ 문서 생성 프롬프트

자주 묻는 질문(FAQ) 문서를 생성하는 프롬프트 템플릿입니다.

---

## 기본 프롬프트

가장 간단한 형태의 프롬프트입니다.

```
raw_data 폴더에 있는 회의록과 슬랙 로그를 분석해서
FAQ 문서를 만들어줘.

자주 나온 질문, 반복 논의된 주제를 기반으로
Q&A 형식으로 정리해줘.

output/faq.md로 저장해줘.
```

---

## 고급 프롬프트 (더 상세한 결과)

체계적이고 카테고리별로 구분된 FAQ를 원할 때 사용하세요.

```
raw_data 폴더의 모든 파일(회의록 10개 + 슬랙 로그 3개)을 분석해서
종합 FAQ 문서를 만들어줘.

## 분석 기준
1. 슬랙에서 반복적으로 질문된 내용
2. 회의록에서 여러 번 논의된 주제
3. 신입사원이 궁금해할 만한 정보
4. 업무 진행 시 자주 발생하는 의문점

## FAQ 구조

### 카테고리별 구분
1. **회사 일반** - 회사, 제품, 조직에 대한 질문
2. **업무 프로세스** - 일하는 방식에 대한 질문
3. **커뮤니케이션** - 협업, 회의, 보고에 대한 질문
4. **인사/복리후생** - HR 관련 질문
5. **기술/제품** - 제품, 개발 관련 질문

### 각 Q&A 형식
- Q: 명확하고 구체적인 질문
- A: 상세하고 실용적인 답변
- 관련 맥락이나 참고 정보 포함

## 작성 가이드
- 실제로 자주 물어볼 법한 질문으로 작성
- 답변은 구체적인 예시나 절차 포함
- 필요시 관련 담당자나 채널 안내

output/faq.md로 저장해줘.
```

## 예상 결과물 형식

Claude Code가 생성하는 FAQ 문서의 예상 형식입니다.

```markdown
# TechStartup Inc. FAQ

> 자주 묻는 질문과 답변을 정리했습니다.
> 궁금한 내용이 없다면 #general 채널에 질문해주세요!

---

## 1. 회사 일반

### Q: 우리 회사의 주요 제품은 무엇인가요?
**A:** TaskFlow라는 프로젝트 관리 SaaS 플랫폼입니다.
팀 협업, 일정 관리, 실시간 커뮤니케이션 기능을 제공합니다.

현재 버전:
- TaskFlow 2.0: 실시간 협업 기능 (2024년 7월 출시)
- TaskFlow 3.0: AI 기능 개발 중 (2024년 10월 예정)

### Q: 현재 고객 규모는 어느 정도인가요?
**A:** 2024년 9월 기준:
- MAU(월간 활성 사용자): 10,000명 이상
- 유료 고객사: 70개 이상
- NPS 점수: 52점

---

## 2. 업무 프로세스

### Q: 채용 프로세스는 어떻게 되나요?
**A:** 2024년 5월 개선된 프로세스입니다 (총 1주일 소요):

1. **서류 전형** (2-3일)
2. **원데이 면접** (하루에 모두 진행)
   - 기술 면접 (1시간)
   - 라이브 코딩 (2시간)
   - 컬처핏 면접 (30분)
3. **최종 결정** (2-3일)

담당자: HR팀 오수현

### Q: 제품 출시 프로세스는?
**A:** 기획 → 개발 → 베타 테스트 → 정식 출시

- 베타 테스트: 프리미엄 고객 10개사 대상
- 베타 기간: 약 1개월
- 담당: 제품기획팀 이서연

---

## 3. 커뮤니케이션

### Q: 슬랙 채널은 어떻게 사용하나요?
**A:** 주요 채널:

| 채널 | 용도 | 주의사항 |
|------|------|----------|
| #general | 전사 공지, 일상 대화 | 중요 공지는 여기서 확인 |
| #dev-issues | 개발 이슈 논의 | 기술 질문은 이 채널에서 |
| #sales-cs | 영업/CS 논의 | 고객 관련 이슈 공유 |

### Q: 긴급한 이슈가 생기면?
**A:**
1. 해당 채널에 메시지 + 담당자 멘션
2. 심각한 경우 DM으로 직접 연락
3. 서버 장애 등 긴급 상황: #general에 공지 + CEO 보고

---

## 4. 인사/복리후생

### Q: 휴가는 어떻게 신청하나요?
**A:**
- 연차: HR팀에 사전 신청
- 여름휴가: 매년 6월 중 일정 제출 (7-8월 사용)
- 병가: 당일 연락 후 사후 서류 제출

담당자: HR팀 박지현

### Q: 사내 이벤트는 어떤 것들이 있나요?
**A:**
- **월간 생일파티**: 해당 월 생일자 케이크 파티
- **분기 워크샵**: 회고 + 전략 공유 + 팀빌딩
- **특별 이벤트**: 플라워 프라이데이(조기퇴근), 할로윈 등
```

## 커스터마이징 팁

### 1. 질문 개수 조절
```
"각 카테고리당 5개씩, 총 25개의 Q&A"
"가장 중요한 10개 질문만"
"최대한 많이, 30개 이상"
```

### 2. 답변 깊이 조절
```
- "답변은 한두 문장으로 간략하게"
- "각 답변에 구체적인 예시와 절차 포함"
- "필요시 관련 문서 링크 안내"
```

### 3. 형식 지정
```
- "표 형식으로 정리"
- "번호 없이 질문만 나열"
- "카테고리 없이 가나다순 정렬"
```

### 4. 특정 주제 집중
```
- "보안 관련 FAQ만"
- "원격 근무 관련 질문만"
- "신규 기능 관련 질문만"
```

### 5. 맥락 추가
```
- "각 답변에 '왜 그런지' 배경 설명 포함"
- "관련된 회의록 날짜 표기"
- "누가 결정했는지 의사결정권자 명시"
```

## 실전 예시

### 예시 1: 간결한 Quick FAQ
```
raw_data를 분석해서 Quick FAQ를 만들어줘.

- 가장 자주 물어볼 10가지 질문만
- 답변은 각각 2문장 이내
- 더 알고 싶으면 누구에게 물어봐야 하는지만 안내

output/quick_faq.md로 저장해줘.
```

### 예시 2: 상황별 FAQ
```
raw_data를 분석해서 상황별 FAQ를 만들어줘.

상황 카테고리:
- "처음 출근했을 때"
- "프로젝트에 투입됐을 때"
- "문제가 생겼을 때"
- "휴가/경조사가 있을 때"

각 상황에서 필요한 정보를 Q&A로 정리해줘.

output/situational_faq.md로 저장해줘.
```

### 예시 3: 인터랙티브 스타일
```
raw_data를 분석해서 FAQ를 만들어줘.

각 답변 끝에 "더 궁금한 점이 있으신가요?" 같은
후속 질문을 안내해줘.

output/chatbot_faq.md로 저장해줘.
```
