## 2. 우리 제품: TaskFlow

### TaskFlow란?
팀의 프로젝트와 태스크를 관리하는 협업 도구입니다.

### 주요 기능
| 버전 | 주요 기능 |
|------|-----------|
| **2.0** (2024.03) | 칸반보드, 간트차트, UI 전면 개편 |
| **2.0** (2024.07) | 실시간 협업, Presence 기능, 동시 편집 |
| **3.0** (2024.10) | AI 기반 일정 추천, 스마트 검색, 자동 리마인더 고도화 |

> "칸반보드 좋다는 피드백 많음. 간트차트 보고 바로 결정하셨어요."
> — 영업/CS 슬랙 채널 (2024-04)

### 기술 스택
- **프론트엔드**: React 18 + Redux Toolkit
- **백엔드**: Node.js
- **실시간 통신**: Socket.io + Redis Pub/Sub
- **배포**: AWS EC2
- **모니터링**: Grafana

> "React 18 업그레이드 + Redux Toolkit 전환 고려 중인데 의견 있으신가요?"
> — 김철수 개발팀장 (#dev 슬랙, 2024-06)

### 요금제 구조
| 플랜 | 가격 | 특징 |
|------|------|------|
| **Free** | 무료 | 5명까지, 히스토리 30일 |
| **Pro** | 월 15,000원 | 10명까지, 기본 기능 |
| **Business** | 월 50,000원 (연간 40,000원) | 무제한 팀원, 고급 권한, 우선 지원 |
| **Enterprise** | 협의 | 전담 지원, E2E 암호화 |

> "Pro와 Enterprise 사이 갭이 너무 커요. 50-200명 규모 회사들이 이탈하고 있어요."
> — 윤재석 영업팀장 (2024-06 가격정책 회의)

## 4. 업무 프로세스

### 요청 관리 시스템
1. **TaskFlow 내부 워크스페이스**에서 "요청 관리" 보드 사용
2. 워크플로우: `요청 → 검토 → 승인 → 진행 → 완료`
3. 개발 승인된 건은 JIRA로 자동 연동

> "요청 사항이 여러 채널로 들어와서 혼란스러워요. Slack으로 오기도 하고, 이메일로 오기도 하고..."
> — 이서연 제품기획팀장 (2024-09 워크플로우 회의)

### 회의 문화
- **No Meeting Wednesday Morning**: 수요일 9-12시 회의 금지
- 회의 요청 시 필수 기재: 아젠다, 예상 결과물, 필수 참석자
- 30분 이내 해결 가능한 건은 Slack 스레드로 대체

> "회의가 너무 많다"는 의견이 많았어요. 하루에 회의만 3-4개씩 있는 날도 있더라고요.
> — 오수현 HR매니저 (2024-09 워크플로우 회의)

### 주간 회의
- **월요일 전체 회의**: 30분, 질의응답 + 의사결정 중심
- 회의 전 **금요일까지** Notion에 주간 업데이트 작성 필수

### OKR (Objectives and Key Results)
- 2024년 Q4부터 도입
- 전사 OKR 3개 + 팀별 OKR 3개
- 월 1회 체크인 미팅

## 5. 개발팀 온보딩 (개발자용)

### 첫 주 일정
1. **Day 1-2**: 코드베이스 파악
2. **Day 3-5**: 개발 환경 셋업 및 간단한 버그 수정
3. **Week 2~**: 본격적인 태스크 시작

> "환영해요! 온보딩 문서 공유할게요. 이번 주는 코드베이스 파악하시고 다음 주부터 간단한 태스크 드릴게요!"
> — 김철수 개발팀장 (#dev 슬랙, 2024-03-25)

### 코드 리뷰 문화
- **모든 PR은 리뷰 필수**
- 리뷰어 최소 1명 승인 후 머지
- 피드백은 구체적으로 (예: "useEffect 의존성 배열 확인해주세요")

> "리뷰 완료! 몇 가지 코멘트 남겼어요. 특히 useEffect 의존성 배열 확인해주세요."
> — 이지영 (#dev 슬랙, 2024-01)

### 배포 프로세스
- **정기 점검**: 새벽 2-4시 (사전 공지 필수)
- **핫픽스**: 긴급 시 즉시 배포 가능
- **QA 주간**: 메이저 버전 출시 전 1주일 집중 테스트

### 장애 대응
1. PagerDuty 알림 확인
2. 원인 파악 및 팀 공유
3. 핫픽스 배포
4. 포스트모템 작성

> "원인 파악! 어제 배포한 코드에서 N+1 쿼리 이슈 발생. 핫픽스 배포합니다."
> — 김철수 개발팀장 (#dev 슬랙, 2024-03-04)

## 6. 복지 및 문화

### 주요 복지
- **플라워 프라이데이**: 분기별 조기 퇴근 (5시 이후 자유)
- **생일 축하**: 케이크 + 팀 점심
- **명절 선물**: 한우세트 등 푸짐한 선물
- **워크샵**: 분기별 팀빌딩 + 회식
- **레퍼럴 보너스**: 시니어 채용 시 200만원

> "4월 플라워 프라이데이! 이번 주 금요일(4/19) 조기 퇴근 가능합니다~ 5시 이후 자유롭게 퇴근하세요."
> — 박지현 HR (#general 슬랙, 2024-04)

### 휴가
- **여름 휴가**: 7-8월 중 신청 (6월 15일까지 제출)
- **연말 연휴**: 보통 12/23~1/1
- **공휴일**: 정상 휴무 (명절 연휴 포함)

### 연말 행사
- **송년회**: 12월 셋째 주 금요일, 인당 15만원 예산
- **고객 감사 이메일**: 12월 중순 발송

> "올해 고생 많았으니 좀 푸짐하게 하죠. 예산은 15만원까지 괜찮아요. 와인도 좀 넉넉히 준비하고요."
> — 김민수 CEO (2024-10 연말계획 회의)
